# 자율주행 탐사로봇 프로젝트 인수인계 v2

## 새 채팅에서 시작할 첫 메시지 (그대로 복사해서 사용)

---

안녕! ROS2 기반 자율주행 탐사로봇을 만들고 있어. 아두이노 펌웨어(모터+엔코더+IMU)는 다 완성하고 검증했고, 이제 라즈베리파이 ROS2 단계로 넘어가려는 참이야. 아래 인수인계 보고 라즈베리파이 ROS2 환경 세팅부터 도와줘. 한 번에 다 하려 하지 말고 한 단계씩 검증하면서 가자.

---

## 1. 프로젝트 목표

ROS2 기반 자율주행 탐사로봇 제작

### 최종 목표
1. 로봇청소기처럼 자율주행하며 맵핑 (SLAM)
2. 장애물 회피
3. 특정 물체 발견 시 사진 찍어서 관제로 전송
4. Unity 관제 UI (배터리, 카메라, 수동조작, 비상정지, 다시시작, 로그, 실시간 위치, 발견지점 표시, 속도조절, 홈버튼)
5. (추후) 음성인식 명령

### 학습 배경
- 학원에서 ROS2 + TurtleBot3 SLAM/Nav2 학습 완료
- 이를 자체 제작 로봇에 적용하는 단계

## 2. 하드웨어 구성 (★ 보드 교체됨)

### 제어 보드
- **아두이노 Nano 33 BLE Sense** (★ 변경: 기존 Nano Every에서 교체)
  - **3.3V 보드. 절대 5V 신호를 핀에 넣지 말 것 (5V tolerant 아님, 보드 손상됨)**
  - IMU 내장: **LSM9DS1** (9축) + 환경센서(LPS22HB 기압, HTS221 온습도, APDS9960 근접)
  - → 별도 IMU 모듈 불필요 (기존 BNO055는 불량으로 폐기)
- 라즈베리파이 4 (ROS2 Humble 예정, **아직 미설치**)

### 구동부
- 뒷바퀴 2개: 엔코더 TT 모터 (6V 160RPM 120:1)
- 앞바퀴 2개: 일반 TT 모터 (엔코더 없음)
- 4WD 차동구동, 좌·우 그룹 병렬

### 모터드라이버
- L298N (ENA·ENB 점퍼캡 꽂아 풀스피드, PWM 안 씀)

### 센서
- RPLIDAR A1M8-R6 (12m 360° LiDAR) — 아직 미연결
- 라즈베리파이 카메라 V2 (8MP) — 아직 미연결

### 전원
- 라즈베리파이: 보조배터리 10000mAh (USB)
- L298N 모터: AA 1.5V × **6개** (= 약 9V, 측정값 8.6~8.7V)

## 3. 확정된 배선 (전부 검증 완료 ✅)

### 모터 (L298N)
| 항목 | 핀 |
|---|---|
| L298N IN1 (좌 방향 1) | D2 |
| L298N IN2 (좌 방향 2) | D3 |
| L298N IN3 (우 방향 1) | D4 |
| L298N IN4 (우 방향 2) | D5 |
| L298N ENA, ENB | 점퍼캡 꽂음 (풀스피드) |
| 아두이노 GND ↔ L298N GND | 공통 연결 필수 |
| L298N 전원입력 +12V/GND | 배터리 9V |

- 좌측 모터 2개: OUT1/OUT2에 병렬 (빨강 2가닥 OUT1, 검정 2가닥 OUT2)
- 우측 모터 2개: OUT3/OUT4에 병렬

### 엔코더
| 항목 | 핀 |
|---|---|
| 좌 엔코더 A상 (초록) | D6 |
| 좌 엔코더 B상 (파랑) | D7 |
| 우 엔코더 A상 (초록) | D8 |
| 우 엔코더 B상 (파랑) | D9 |
| 엔코더 VCC | **아두이노 3.3V** (★ 5V 아님! 신호 5V 되면 보드 손상) |
| 엔코더 GND | 아두이노 GND |

- 엔코더 신호선 색: A상=초록, B상=파랑, VCC=빨강, GND=검정

### IMU
- 내장이라 배선 없음. 라이브러리만 설치.

## 4. 통신 프로토콜 (115200 baud)

### Pi → 아두이노
```
M,<leftDir>,<rightDir>\n    -1 후진 / 0 정지 / 1 전진
```

### 아두이노 → Pi
```
R,imu_ok 또는 R,imu_not_connected   IMU 초기화 결과
R,ready                            준비 완료
E,<lcnt>,<rcnt>                     엔코더 카운트 (50Hz)
I,ax,ay,az,gx,gy,gz                IMU 가속도+자이로 (50Hz)
Mag,mx,my,mz                       지자계 (10Hz)
S,timeout                          0.5초 명령 없으면 자동 정지
```

### 방향 정의 (배선 기준 보정 완료)
- `M,1,1` = 전진, `M,-1,-1` = 후진, `M,1,-1` = 제자리 회전
- setMotor 함수에서 HIGH/LOW 반전으로 방향 맞춤 (양쪽 다)

## 5. 완성된 펌웨어

`base_controller.ino` (Nano 33 BLE Sense용, 첨부 파일)
- 모터 명령 수신 + 엔코더/IMU/지자계 송신 통합
- 50Hz 엔코더·IMU, 10Hz 지자계
- 0.5초 명령 타임아웃 안전정지
- IMU 미연결 시에도 동작 (fallback)
- 라이브러리: `Arduino_LSM9DS1` 필요
  - (만약 보드가 Rev2면 `Arduino_BMI270_BMM150`로 교체. 현재는 LSM9DS1 = Rev1)

### IMU 설계 결정
- 아두이노는 **raw 데이터(가속도/자이로/지자계)만** 송신
- 센서 융합(방향 계산)은 라즈베리파이 ROS의 `imu_filter_madgwick`에서 (use_mag 옵션으로 지자계 활용 → yaw drift 보정)

## 6. 진행 상황

### 완료 ✅
1. L298N + 모터 + 전원 + 점퍼캡 — 손 테스트로 검증
2. 아두이노 → L298N 신호 제어 (3.3V 신호로 모터 구동 확인)
3. 엔코더 좌·우 독립 카운트 (3.3V 구동, 신호도 3.3V 확인)
4. 내장 IMU + 환경센서 전부 동작
5. 통합 펌웨어 (모터+엔코더+IMU+지자계) + 시리얼 프로토콜
6. 모터 방향 보정 (M,1,1 = 전진)

### 다음 단계 (미시작)
5단계. **라즈베리파이 ROS2 환경 세팅** ← 여기서 시작
  - ROS2 Humble 설치 (아직 미설치)
  - 워크스페이스 생성
6단계. **base_controller 노드 작성** — 시리얼 ↔ ROS2 토픽
  - 구독: `/cmd_vel` → M 명령으로 변환
  - 발행: `/odom` (엔코더 기반), `/imu/data_raw`, `/imu/mag`
7단계. URDF (로봇 형상 정의)
8단계. RPLIDAR 드라이버 + SLAM
9단계. Nav2 (자율주행)
10단계. 카메라 + 물체 발견 시 촬영·전송
11단계. Unity 관제 UI

## 8. 그동안의 핵심 교훈 (디버깅 노트)

1. **한 번에 한 단계씩, 각 단계 멀티미터/시리얼로 검증하고 넘어간다.** 이게 제일 중요.
2. **배선 시작 전 그림으로 정리하고 확인.**
3. 모터가 안 돌 때 흔한 원인 순서: ① ENA·ENB 점퍼 없음/빠짐 → ② GND 공통 연결 누락 → ③ 신호선 위치/접촉 → ④ 전원.
4. **모터 안 돌면 "손 테스트"로 분리 진단** (아두이노 빼고 IN1→5V, IN2→GND 직접 꽂아 모터만 확인). L298N/모터 문제냐 아두이노 연결 문제냐를 한 방에 가름.
5. **시리얼은 뜨는데 모터만 안 돈다** = 코드/신호는 정상, ENA나 GND 공통 의심.
6. 보드 교체 시 모든 선을 다시 꽂아야 하므로 1~3단계 재검증 필수.
7. **3.3V 보드(Nano 33 BLE Sense)는 5V 신호 절대 금지.** 엔코더 등 5V 출력 센서는 3.3V로 구동하거나 레벨시프터 사용.
8. BNO055(외부 IMU)는 클론 보드 불량 잦고 I2C 까다로움. 진단법: 모듈 떼면 SDA 라인 살아나고 꽂으면 0V로 죽으면 모듈 불량. → 내장 IMU 보드로 전환하여 해결.
