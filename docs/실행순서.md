# 실행 순서 (SLAM 매핑)

> 분산 구조: **로봇(Pi)** 에서 센서·모터, **데스크탑** 에서 SLAM·rviz·운전.
> 창마다 별도 터미널. 로봇 창(1·2)은 `ssh`로 접속.
> 켜는 순서: **로봇(1·2) 먼저 → 데스크탑(3·4·5)**

## 🤖 로봇(Pi) — `ssh`로 접속

### 창 1 — 로봇 몸체 (바퀴·위치·IMU)
```bash
ssh aeri@192.168.75.100
cd ~/ros2_ws && source install/setup.bash
ros2 launch autonomous_robot robot.launch.py
```
→ `Serial opened` 나오면 성공.

### 창 2 — 라이다
```bash
ssh aeri@192.168.75.100
sudo chmod 666 /dev/ttyUSB0
ros2 launch rplidar_ros rplidar.launch.py serial_port:=/dev/ttyUSB0 serial_baudrate:=115200 frame_id:=laser
```
→ 라이다가 돌면 성공.

## 🖥️ 데스크탑 — `ssh` 없이 그대로

### 창 3 — SLAM (지도 그리기)
```bash
source /opt/ros/jazzy/setup.bash
source ~/autonomous-exploration-robot/ros2_ws/install/setup.bash
ros2 launch autonomous_robot slam.launch.py
```

### 창 4 — rviz (지도 화면)
```bash
source /opt/ros/jazzy/setup.bash
rviz2
```
→ rviz 안: **Fixed Frame = `map`** / **Add → Map(`/map`), LaserScan(`/scan`)**

### 창 5 — teleop (키보드 운전)
```bash
source /opt/ros/jazzy/setup.bash
source ~/autonomous-exploration-robot/ros2_ws/install/setup.bash
ros2 run autonomous_robot teleop_node
```
→ 운전: `w`전진 `x`후진 `a`좌회전 `d`우회전 `s`정지 (이 창을 클릭한 상태에서).
**천천히**: 조금 가고 → `s`로 멈추고 → 1~2초 기다리기. 빠른 회전 금지.

## 💾 지도 저장 (아무 데스크탑 터미널)
```bash
source /opt/ros/jazzy/setup.bash
ros2 run nav2_map_server map_saver_cli -f ~/maps/원하는이름
```
→ `~/maps/원하는이름.pgm` + `.yaml` 생성 (확장자는 안 붙임). SLAM 켜진 동안 언제든 가능.

## 🔄 지도 리셋·SLAM 재시작 (창 3에서)
```
Ctrl + C  →  ↑(위 화살표)  →  엔터
```

## 📝 참고
- 창 끄기: 각 창에서 `Ctrl + C`. **로봇 창(1·2)에서 `exit` 치면 로봇 프로그램이 같이 꺼지니** 주의.
- 로봇 IP `192.168.75.100`은 DHCP라 공유기에 따라 바뀔 수 있음. 안 되면 데스크탑에서 `ros2 topic list`로 로봇 토픽이 보이는지 확인.
- `ROS_DOMAIN_ID=0`은 양쪽 `~/.bashrc`에 등록돼 있어 매번 안 쳐도 됨. (안 되면 `export ROS_DOMAIN_ID=0`)
- 프롬프트로 구분: `aeri@aeriPC` = 로봇 / `aeri@aeri` = 데스크탑.
